# Реляційні бази даних

## Що таке реляційна БД?

БД, де дані зберігаються в таблицях (відношеннях).

## Структура

Таблиця (Table) = Відношення (Relation)
Рядок (Row) = Запис (Record) = Кортеж (Tuple)
Стовпець (Column) = Поле (Field) = Атрибут (Attribute)

## Основні характеристики реляційних БД

Характеристика - Опис:

Табличний формат:
Дані в рядках і стовпцях

Унікальні рядки:
Кожен рядок унікальний (PK)

Атомарні значення:
1 комірка = 1 значення

Порядок не важливий:
Рядки/стовпці не мають порядку

Зв'язані через ключі:
Таблиці незалежні, зв'язані через FK

SQL:
Стандартна мова запитів

ACID:
Гарантії транзакцій

## Особливості організації даних

### 1. Первинний ключ (Primary Key)
Унікально ідентифікує кожен рядок.

### 2. Зовнішній ключ (FK)
Зв'язок між таблицями.

### 3. Нормалізація
Дані розділені на таблиці, щоб уникати дублів.

## Особливості зберігання даних

Файли:
Дані на диску в спеціальних файлах

Сторінки:
Дані розділені на блоки (4-16кБ)

Індекси:
Окремі структури для швидкого пошуку

Журнал (Log):
Запис всіх змін для відновлення

Кеш:
Часто використовувані дані в пам'яті

## DBMS (Database Management System)

Програмне забезпечення для керування БД.

Інкапсуляція (СКБД)

## Функції DBMS

Що робить:

Зберігання:
Організує дані на диску

Запити:
Виконує SQL (SELECT, INSERT)

Безпека:
Контроль доступу, авторизація

Цілісність:
Перевірка правил (constraints)

Транзакції:
ACID гарантії

Конкурентність:
Багато користувачів одночасно

## Популярні реляційні DBMS

DBMS - Тип - Використання:

MySQL:
Open Source - Веб, стартапи

PostgreSQL:
Open Source - Складні проєкти, аналітика

Oracle:
Комерційна - Enterprise, банки

MS SQL Server:
Комерційна - Enterprise, Windows

SQLite:
Вбудована - Моб. додатки

MariaDB:
Open Source - Форк MySQL

## Архітектура DBMS

Користувач
    ↓ SQL запити
Query Processor (парсинг, оптимізація, план)
    ↓
Storage Engine (читання, запис, індекси, кеш)
    ↓
Disk (файли)

## Переваги реляційних БД

Схема обов'язкова:
Чітка структура даних

Constraints для політики даних:
Правила цілісності

Стандартна потужна мова:
SQL

Підтримка транзакцій:
ACID гарантії

40+ років розвитку, стабільно:
Перевірені часом технології

## Недоліки реляційних БД

Масштабування горизонтальне:
Складне (sharding)

Повільне на великих даних:
При величезних обсягах

Жорстка схема:
Важко змінювати структуру

Гірше для документів:
Краще NoSQL для неструктурованих даних
