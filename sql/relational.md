# Реляційні бази даних

## Що таке реляційна БД?

БД, де дані зберігаються в таблицях (відношеннях).

---

## Структура

| Термін | Синоніми |
|---|---|
| Таблиця (Table) | Відношення (Relation) |
| Рядок (Row) | Запис (Record), Кортеж (Tuple) |
| Стовпець (Column) | Поле (Field), Атрибут (Attribute) |

---

## Основні характеристики

| Характеристика | Опис |
|---|---|
| **Табличний формат** | Дані в рядках і стовпцях |
| **Унікальні рядки** | Кожен рядок унікальний (PK) |
| **Атомарні значення** | 1 комірка = 1 значення |
| **Порядок не важливий** | Рядки/стовпці не мають порядку |
| **Зв'язані через ключі** | Таблиці незалежні, зв'язані через FK |
| **SQL** | Стандартна мова запитів |
| **ACID** | Гарантії транзакцій |

---

## Особливості організації даних

### 1. Первинний ключ (Primary Key)
Унікально ідентифікує кожен рядок таблиці.

### 2. Зовнішній ключ (Foreign Key)
Зв'язок між таблицями — значення в одній таблиці посилається на PK іншої.

### 3. Нормалізація
Дані розділені на таблиці, щоб уникати дублювання.

---

## Особливості зберігання даних

| Компонент | Опис |
|---|---|
| **Файли** | Дані на диску в спеціальних файлах |
| **Сторінки** | Дані розділені на блоки (4–16 кБ) |
| **Індекси** | Окремі структури для швидкого пошуку |
| **Журнал (Log)** | Запис всіх змін для відновлення |
| **Кеш** | Часто використовувані дані в пам'яті |

---

## DBMS (Database Management System)

Програмне забезпечення для керування БД. Інкапсулює всю логіку роботи з даними.

### Функції DBMS

| Функція | Опис |
|---|---|
| **Зберігання** | Організує дані на диску |
| **Запити** | Виконує SQL (SELECT, INSERT тощо) |
| **Безпека** | Контроль доступу, авторизація |
| **Цілісність** | Перевірка правил (constraints) |
| **Транзакції** | ACID гарантії |
| **Конкурентність** | Багато користувачів одночасно |

### Популярні реляційні DBMS

| DBMS | Тип | Використання |
|---|---|---|
| **MySQL** | Open Source | Веб, стартапи |
| **PostgreSQL** | Open Source | Складні проєкти, аналітика |
| **Oracle** | Комерційна | Enterprise, банки |
| **MS SQL Server** | Комерційна | Enterprise, Windows |
| **SQLite** | Вбудована | Мобільні додатки |
| **MariaDB** | Open Source | Форк MySQL |

---

## Архітектура DBMS

```
Користувач
    ↓ SQL запити
Query Processor (парсинг, оптимізація, план виконання)
    ↓
Storage Engine (читання, запис, індекси, кеш)
    ↓
Disk (файли)
```

---

## Переваги реляційних БД

| Перевага | Опис |
|---|---|
| **Обов'язкова схема** | Чітка структура даних |
| **Constraints** | Правила цілісності даних |
| **SQL** | Стандартна потужна мова запитів |
| **ACID** | Гарантії транзакцій |
| **Зрілість** | 40+ років розвитку, перевірені часом |

---

## Недоліки реляційних БД

| Недолік | Опис |
|---|---|
| **Горизонтальне масштабування** | Складне (sharding важко реалізувати) |
| **Великі обсяги даних** | Повільніше при величезних датасетах |
| **Жорстка схема** | Важко змінювати структуру |
| **Неструктуровані дані** | Краще підходить NoSQL |
